PB Solver CPU Optimization — 383 s → 81 s (4.7×)

Three optimization layers applied to the pure-Python nonlinear Poisson–Boltzmann solver, each validated on the cal_18 reference case against the frozen accuracy gate. The final configuration beats the VASP/CEP-DIP native solve (180.7 s wall) by 2.2× while improving every accuracy metric. All runs on one gpu-a100-dev node (32 threads), same solver parameters as the baseline (tol 1e-3, max-outer 12, cg-max-iter 40).

Where the Time Went, Config by Config

Stage-stacked wall time. Step 1 fuses all hot elementwise loops into OpenMP C kernels (the profiled single-thread NumPy fraction) and reuses accepted line-search field quantities. Step 2 warm-starts fixstep 0 from a half-resolution Newton solve prolonged spectrally. Step 3 switches the whole PB pipeline to half-spectrum r2c/c2r transforms.

Accuracy Gate (cal_18, final fixstep)

Gate thresholds are the retained baseline values — a change may not regress them. Step 1 reproduces the baseline trajectory to 1e-12 (identical arithmetic, reordered). Steps 2–3 take a different (legitimate) Newton path within the residual tolerance; step 3 lands better on every metric.

What Was Changed

  • Fused OpenMP kernels (step 1, 1.9×). Profiling showed 32→125 cores gave zero speedup: the time sat in single-threaded NumPy elementwise chains, not FFTs. Sixteen C kernels (_pb_fast.c) now stream each 87–165 MB operand once, in parallel: the dielectric tensor contraction, χ response, ion density, Langevin g(y), complex kernel multiplies, and the whole CG update algebra. Newton now also reuses the accepted line-search trial's field quantities. Fused vs NumPy paths agree to 4e-15; PB_DISABLE_FUSED=1 restores the reference path.
  • Coarse-grid warm start (step 2, +1.4×). Fixstep 0 burned 99 CG iterations converging from φ=0. The same Newton solve on the half-resolution grid (8× cheaper per iteration) followed by Fourier zero-padding prolongation leaves the fine grid only 3 outer iterations. --coarse-init, on by default for even grids.
  • Half-spectrum r2c/c2r (step 3, +2.3×). All real fields; the full complex spectrum was redundant. PB_RFFT=1 halves FFT work and spectral memory traffic. Inner products get mode-multiplicity weights; derivative premultipliers zero each lattice axis's Nyquist frequency before combining into cartesian components — this reproduces the full-spectrum ifft(...).real semantics while preserving the cross terms of non-orthogonal cells.
Found along the way: for non-orthogonal cells the original full-spectrum gsq is asymmetric on Nyquist planes (a pre-existing property inherited from the FFT index convention; harmless because physical fields carry no Nyquist-scale content). The two spectral representations are equally valid discretizations that differ only there — which is why steps 2–3 are validated end-to-end rather than step-by-step.

Recommendation

Merge perf/cpu-fused-kernels into main with PB_RFFT=1 and --coarse-init as the default configuration for production solves. Remaining headroom: read-input caching (~10 s), and a CuPy/GPU port (the pipeline is now cleanly FFT + fused kernels; est. 10×+ on the PB core) — neither attempted in this round.
